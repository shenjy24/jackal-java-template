# 离线软件安装文档

本文档说明如何在无法访问公网的服务器上安装 Git、Docker Engine、Docker Compose，并创建可执行 Docker 操作的 `tech` 用户。

## 目录结构

将 `document/deploy/software/offline/` 下的文件上传到服务器，例如：

```text
software-offline/
├── install-software.sh
└── pkgs/
    ├── git-binaries.linux-64bit.tar.gz
    ├── docker-29.5.3.tgz
    └── docker-compose-linux-x86_64
```

`pkgs` 是推荐目录名，表示离线安装包集合。也可以通过 `PKG_DIR` 指定其他目录。

## 安装

进入离线安装目录：

```bash
cd software-offline
chmod +x install-software.sh
sudo bash install-software.sh
```

脚本会执行：

1. 创建 `tech` 用户，已存在则跳过。
2. 将 `tech` 加入 `sudo`、`wheel` 和 `docker` 组。
3. 解压并安装 Git。
4. 解压并安装 Docker Engine，并创建 `containerd.service`、`docker.service`。
5. 安装 Docker Compose 到 Docker CLI 插件目录。

如果安装包名称不同：

```bash
sudo GIT_ARCHIVE=git.tar.gz \
  DOCKER_ARCHIVE=docker.tgz \
  DOCKER_COMPOSE_BINARY=docker-compose-linux-x86_64 \
  bash install-software.sh
```

如果安装包目录不同：

```bash
sudo PKG_DIR=/data/offline-pkgs bash install-software.sh
```

## 验证

```bash
git --version
docker --version
docker compose version
systemctl status docker
```

如果 `tech` 用户刚被加入 `docker` 组，需要重新登录该用户后，用户组权限才会生效：

```bash
su - tech
docker ps
```

## 注意事项

- `docker-29.5.3.tgz` 需要是 Docker 官方 Linux 静态二进制包，解压后应包含 `docker/` 目录。
- `docker-compose-linux-x86_64` 需要与服务器 CPU 架构一致。
- `git-binaries.linux-64bit.tar.gz` 需要解压后在顶层包含 `git*` 可执行文件。脚本会复制到 `/usr/local/bin/` 并赋予执行权限。
- 服务器需具备 `tar`、`find`、`groupadd`、`useradd`、`usermod`、`systemctl` 等基础系统命令。
