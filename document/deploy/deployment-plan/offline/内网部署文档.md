# 内网部署文档

本文档说明服务器无法访问公网时，如何采用“上传 jar、内网本机构建镜像、启动容器”的方式部署 `jackal-java-template`。

## 方案概览

内网服务器不拉代码、不下载 Maven 依赖，只做以下动作：

1. 首次准备：安装 Docker 和 Docker Compose Plugin。
2. 首次准备：加载固定基础镜像 tar，例如 `mysql:8.4`、`eclipse-temurin:21-jre`，可选 `nginx:1.30.2`。
3. 每次发布：上传新 jar 到服务器。
4. 每次发布：用内网部署目录下的 `Dockerfile` 在内网服务器构建应用镜像。
5. 每次发布：用 `docker compose up -d --no-build app-server` 启动或替换应用容器。

默认部署参数：

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `ENV` | `prod` | Spring Profile |
| `REPO_NAME` | `java-template` | 应用镜像仓库名和容器名前缀 |
| `APP_PORT` | `18080` | 宿主机应用端口，映射到容器 `8080` |
| `APP_LOG_DIR` | `../logs` | 应用日志目录，相对 `deploy` 目录解析 |
| `APP_MEMORY_LIMIT` | `1G` | 应用容器内存上限 |
| `APP_MEMORY_RESERVATION` | `512M` | 应用容器内存预留 |
| `IMAGE_KEEP_COUNT` | `3` | 保留最近应用镜像数量 |
| `RUNTIME_IMAGE` | `eclipse-temurin:21-jre` | 构建应用镜像使用的 Java 运行时基础镜像 |
| `JAR_DIR` | `../jars` | 上传 jar 目录，相对 `deploy` 目录解析 |

## 目录说明

```text
deploy/
├── 内网部署文档.md
├── Dockerfile                  # 使用上传的 jar 构建应用镜像
├── docker-compose.yml          # 启动和替换应用容器
├── start.sh                    # 根据 jar 构建应用镜像并启动应用容器
├── rollback.sh                 # 回滚到已存在的历史应用镜像
├── load-images.sh              # 加载 MySQL/JRE/Nginx 等基础镜像 tar
├── deploy.env                  # 应用部署变量
└── images.env                  # 基础镜像加载变量
```

推荐内网服务器目录：

```text
部署根目录/
├── deploy/                     # 本目录由 document/deploy/deployment-plan/offline/ 下的文件复制而来
│   ├── Dockerfile
│   ├── docker-compose.yml
│   ├── deploy.env
│   ├── images.env
│   ├── load-images.sh
│   ├── start.sh
│   └── rollback.sh
├── java-template/              # 业务代码仓库目录，与 deploy 同级
│   └── document/
│       ├── deploy/mysql/
│       ├── deploy/nginx/
│       └── sql/production/
├── jars/
│   └── jackal-java-template-1.0.0-SNAPSHOT.jar
├── middleware-images/
│   ├── mysql-8.4.tar
│   ├── eclipse-temurin-21-jre.tar
│   └── nginx-1.30.2.tar
└── logs/
```

## 基础镜像准备

在可访问 Docker 镜像仓库的机器上准备固定镜像 tar：

```bash
docker pull mysql:8.4
docker pull eclipse-temurin:21-jre

docker save -o mysql-8.4.tar mysql:8.4
docker save -o eclipse-temurin-21-jre.tar eclipse-temurin:21-jre
```

如果内网需要 Nginx 容器，也准备 Nginx 镜像：

```bash
docker pull nginx:1.30.2
docker save -o nginx-1.30.2.tar nginx:1.30.2
```

将镜像 tar 上传到内网服务器：

```text
/data/java-template/middleware-images/
```

## 内网服务器首次准备

内网服务器需提前安装：

- Docker Engine
- Docker Compose Plugin
- `tar` / `gzip`

准备部署目录：

```bash
sudo mkdir -p /data/java-template/deploy /data/java-template/jars /data/java-template/middleware-images /data/java-template/logs
sudo chown -R $USER:$USER /data/java-template
cd /data/java-template
```

验证 Docker：

```bash
docker --version
docker compose version
```

将 `document/deploy/deployment-plan/offline/` 下的文件复制到服务器 `deploy/` 目录。上传后至少需要：

```text
deploy/
java-template/document/deploy/mysql/
java-template/document/deploy/nginx/
java-template/document/sql/production/
```

## 加载基础镜像

按需调整基础镜像配置：

```bash
vim deploy/images.env
```

示例：

```env
MYSQL_IMAGE=mysql:8.4
MYSQL_IMAGE_TAR=../middleware-images/mysql-8.4.tar

LOAD_RUNTIME_IMAGE=1
RUNTIME_IMAGE=eclipse-temurin:21-jre
RUNTIME_IMAGE_TAR=../middleware-images/eclipse-temurin-21-jre.tar

LOAD_NGINX_IMAGE=0
NGINX_IMAGE=nginx:1.30.2
NGINX_IMAGE_TAR=../middleware-images/nginx-1.30.2.tar
```

加载镜像：

```bash
bash deploy/load-images.sh
```

验证：

```bash
docker image inspect mysql:8.4
docker image inspect eclipse-temurin:21-jre
```

如果使用 Nginx：

```bash
docker image inspect nginx:1.30.2
```

## 启动 MySQL

首次启动 MySQL 前，准备初始化 SQL：

```bash
mkdir -p java-template/document/deploy/mysql/mysql/init
cp java-template/document/sql/production/ddl/auth.sql java-template/document/deploy/mysql/mysql/init/01-auth-ddl.sql
cp java-template/document/sql/production/ddl/user.sql java-template/document/deploy/mysql/mysql/init/02-user-ddl.sql
cp java-template/document/sql/production/dml/auth.sql java-template/document/deploy/mysql/mysql/init/03-auth-dml.sql
```

准备 MySQL 环境变量：

```bash
export MYSQL_PORT=3306
export MYSQL_ROOT_PASSWORD='生产数据库密码'
export MYSQL_DATABASE=template
```

启动 MySQL：

```bash
docker compose -f java-template/document/deploy/mysql/docker-compose-mysql.yml up -d
```

注意：MySQL 官方镜像只会在首次创建数据目录时执行 `/docker-entrypoint-initdb.d` 下的 SQL。已经创建过 `mysql-data` 数据卷后，`mysql/init` 下的 SQL 不会重复执行。

## 外网构建 jar

在可联网开发机或构建机执行：

```bash
mvn -q -DskipTests package
```

构建产物位于：

```text
target/jackal-java-template-1.0.0-SNAPSHOT.jar
```

将 jar 上传到内网服务器：

```text
/data/java-template/jars/
```

## 内网构建镜像并启动应用

按需调整应用部署配置：

```bash
vim deploy/deploy.env
```

常用配置：

```env
ENV=prod
REPO_NAME=java-template
APP_PORT=18080
APP_LOG_DIR=../logs
APP_MEMORY_LIMIT=1G
APP_MEMORY_RESERVATION=512M
IMAGE_KEEP_COUNT=3
RUNTIME_IMAGE=eclipse-temurin:21-jre
JAR_DIR=../jars
```

如果 `JAR_DIR` 下只有一个 jar，可以直接执行：

```bash
bash deploy/start.sh
```

如果有多个 jar，显式指定：

```bash
bash deploy/start.sh \
  /data/java-template/jars/jackal-java-template-1.0.0-SNAPSHOT.jar
```

脚本会：

1. 校验 jar 文件存在。
2. 校验 `RUNTIME_IMAGE` 已经存在。
3. 使用内网部署目录下的 `Dockerfile` 构建应用镜像。
4. 写入 `deploy/app-image.env`，供回滚脚本读取。
5. 使用 `deploy/docker-compose.yml` 启动或替换应用容器。
6. 保留最近 `IMAGE_KEEP_COUNT` 个 `java-template:prod-*` 镜像，并清理悬空镜像。

镜像 tag 示例：

```text
java-template:prod-20260608120000-jackal-java-template-1.0.0-SNAPSHOT
```

## 后续发布

每次修复问题或发布新版本：

1. 外网构建新 jar。
2. 上传新 jar 到 `/data/java-template/jars/`。
3. 执行 `start.sh` 构建新应用镜像并重启应用。

示例：

```bash
bash deploy/start.sh \
  /data/java-template/jars/jackal-java-template-1.0.0-SNAPSHOT.jar
```

后续应用发布不需要重复加载 MySQL、JRE、Nginx 镜像，除非基础镜像版本需要升级。

## 回滚

查看本机已有应用镜像：

```bash
docker image ls java-template --format "{{.Repository}}:{{.Tag}} {{.CreatedSince}} {{.Size}}" | grep ":prod-"
```

回滚到指定镜像：

```bash
bash deploy/rollback.sh \
  java-template:prod-20260608120000-jackal-java-template-1.0.0-SNAPSHOT
```

回滚只切换应用镜像，不回滚数据库结构和数据。涉及数据库变更的版本上线前，需要提前准备数据库备份和兼容方案。

回滚只能使用本机仍然存在的镜像；`start.sh` 默认只保留最近 3 个应用镜像。如果需要更长回滚窗口，请调大 `IMAGE_KEEP_COUNT`。

## Nginx

Nginx 在内网方案中是可选项。

不需要域名、HTTPS、前后端统一入口时，可以不部署 Nginx，直接访问：

```text
http://内网服务器IP:18080
```

如果需要 Nginx，建议使用提前加载好的 Nginx 镜像容器化运行，并挂载本项目的参考配置：

```text
java-template/document/deploy/nginx/nginx-prod.conf
```

需要按内网实际情况修改：

- `server_name`
- `ssl_certificate`
- `ssl_certificate_key`
- 如果没有 HTTPS 证书，可临时改为只监听 `80`
- `/api/` 默认转发到 `http://127.0.0.1:18080/`

## MySQL 备份

手动备份：

```bash
cd java-template/document/deploy/mysql
chmod +x mysql-backup.sh
DB_PASS=生产数据库密码 ./mysql-backup.sh
```

建议配置定时备份：

```bash
crontab -e
```

示例：

```cron
30 2 * * * cd /data/java-template/java-template/document/deploy/mysql && DB_PASS='生产数据库密码' /bin/bash mysql-backup.sh >> /dev/null 2>&1
```

## 常见问题

### start.sh 提示基础镜像不存在

先加载 JRE 基础镜像：

```bash
docker load -i /data/java-template/middleware-images/eclipse-temurin-21-jre.tar
```

并确认 tag 与 `deploy.env` 中一致：

```bash
docker image inspect eclipse-temurin:21-jre
```

### docker compose 启动时仍尝试构建

确认启动命令使用的是：

```bash
docker compose -f deploy/docker-compose.yml up -d --no-build app-server
```

并确认 `APP_IMAGE` 指向已经构建好的应用镜像。

### 应用连不上 MySQL

检查：

- MySQL 容器是否已启动。
- `application-prod.yml` 中数据库地址是否能从应用容器访问。
- 使用本仓库默认 MySQL 容器时，应用连接宿主机 MySQL 可使用 `172.17.0.1:3306`。
- 应用配置中的数据库名是否与 `MYSQL_DATABASE` 一致。

### 初始化 SQL 没有执行

MySQL 初始化脚本只在数据卷第一次创建时执行。确认无生产数据后，才可以删除数据卷重建：

```bash
docker compose -f java-template/document/deploy/mysql/docker-compose-mysql.yml down -v
```

生产环境执行该命令会删除数据库数据，必须先备份。
